Befehlszeilen-Optionen

Sie können viele von Logisims Anwendungseinstellungen über Kommandozeilenoptionen konfigurieren. Dies kann besonders in einem Labor mit Computern für einzelne Studenten nützlich sein, wo Sie möchten, dass Logisim für die Studenten jedes Mal das gleiche Programm startet, unabhängig davon, wie die vorherigen Studenten das Programm konfiguriert haben.

Die allgemeine Befehlszeilensyntax lautet wie folgt. 

java -jar jarFileName [options] [filenames]

Die auf der Befehlszeile genannten optionalen Zusatzdateien werden als separate Fenster innerhalb von Logisim geöffnet.

Das folgende Beispiel startet Logisim in seiner Grundkonfiguration. 

java -jar jarFileName --user-template plain -gates ansi -locale en
usage: Logisim-evolution [-b ] [--clear-prefs] [-f ] [-g ] [-h] [-l ] [-m ] [-n ] [--no-splash] [-o ] [-s ] [-t ] [--toplevel-circuit ] [-u ] [-v] [-w ]

Zu den unterstützten Optionen gehören die folgenden.

-u, --user-template [empty|plain|filename]

Konfiguriert die zu verwendende Vorlage für Logisim.

-g, --gates [ansi|iec]

Konfiguriert den zu verwendenden Gate-Typ.

-o, --locale LocaleIdentifier

Konfiguriert, welche Übersetzung verwendet werden soll. Zum Zeitpunkt der Erstellung dieses Dokuments sind die unterstützten Gebietsschemata enthalten:

de Deutsch
el Griechisch
en Englisch
es Spanisch
fr Französisch
pt Portugiesisch
ru Russisch
it Italienisch
nl Niederländisch
jp Japanisch
pl Polnisch
--clear-prefs

Löschen Sie alle Anwendungseinstellungen beim Start, so dass Logisim sich so verhält, als ob es zum ersten Mal auf dem Hostsystem ausgeführt wird.

--no-splash

Blendet den Logisim-Einstiegsbildschirm aus.

-h, --help

Zeigt eine Zusammenfassung der Befehlszeilenoptionen an.

-v, --version

Zeigt die Logisim-Versionsnummer an.

-l, --load filename

Ein Bild in den RAM laden (funktioniert nur mit -tty)

-s, --substitute Datei1 Datei2

laden Sie die Datei, indem Sie die Bibliotheksdatei (Datei1) durch die Bibliotheksdatei (Datei2) ersetzen. Weitere Informationen in Ersetzen von Bibliotheken

-t, --tty [table|speed|tty|halt|stats|binary|hex|csv|tabs]

Ohne GUI ausführen. Weitere Informationen in Andere Überprüfung

-w, --test-vector Schaltungsname Vektordatei Projektdatei

Führen Sie die Tests Schaltungsname in Projektdatei basierend auf den Vektoren in Vektordatei aus. Weitere Informationen in Fenster-Text-Vektor

-m, --geometry [BreitexHöhe|BreitexHöhe+X+Y]

Öffnet Logisim mit einem Fenster, das nach den Parametern W (Breite) und H (Höhe) bemessen ist, oder mit der linken oberen Ecke, die nach den Parametern X,Y positioniert ist.
Beispiel: -m 600x400+100+100

-n, --new-file-format circ_input circ_output

Open a .circ file of any version (circ_input) of Logisim and write a new one (circ_output) in the current format used in Logisim-evolution.
