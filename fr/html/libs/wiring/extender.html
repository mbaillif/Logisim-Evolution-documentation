Extension de bit

Apparence :

Comportement

L'extenseur de bits modifie la largeur de donnée en bit d'une connexion. Si sa largeur est réduite, elle est simplement tronquée pour conserver les bits de poids le plus faible. Si sa largeur de donnée est augmentée, les bits de poids faible ne sont pas modifiés, et vous avez le choix pour les bits supplémentaires de poids fort : ils peuvent êtres soit tous à 0 soit tous à 1 ou tous égale au bit de signe (le bit le plus élevé de l'entrée) ou encore êtres déterminés par une entrée de commande supplémentaire.

Pins

Bord ouest :
L'entrée multibits dont la valeur doit être transformée. La largeur de donnée en bit est définie par l'attribut largeur bus entrée.
Bord Est :
La sortie transformée. La largeur de donnée en bit est définie par l'attribut Largeur bus sortie.
Bord Nord :
Spécifie quels devraient être les bits supplémentaires dans la sortie. Cette broche est disponible uniquement lorsque l'attribut Type d'extension est Entrée Sa largeur de donnée en bit est de 1.

Attributs

Lorsque le composant est sélectionné ou ajouté, Les touches 0 à 9 modifient l'attribut Entrée : Largeur donnée, les touches Alt-0 à Alt-9 modifient l'attribut Sortie : Largeur donnée.

Entrée : Largeur de donnée
Décrit plus haut.
Sortie : Largeur de donnée
Décrit plus haut.
Type d'extension
Dans le cas d'une extension de la largeur, cette option permet de déterminer l'état des bits supplémentaire. Si zéro ou un, les bits supplémentaires seront égale à 0 ou à 1 en conséquence. Si Signe, les bits supplémentaires sont ajustés pour correspondre au bit de poids fort de l’entrée. Et si Entrée, La valeur d'une seconde entrée de un bit sur son côté est utilisée pour définir les bits supplémentaires.
