Options de la ligne de commande

Vous pouvez configurer de nombreuses préférences d'application de Logisim via les options de ligne de commande. Cela peut être particulièrement utile dans un laboratoire d'ordinateurs composés poste d'étude où vous souhaitez que Logisim démarre la même chose pour les étudiants à chaque fois, indépendamment de la configuration du programme fait par l'étudiant précédent.

La syntaxe globale de la ligne de commande est la suivante. 

java -jar jarFileName [options] [filenames]

Les fichiers supplémentaires facultatifs nommés sur la ligne de commande seront ouverts en tant que fenêtres distinctes dans Logisim.

L'exemple suivant démarre Logisim dans sa configuration de base. 

java -jar jarFileName -plain -gates shaped -locale en
usage: Logisim-evolution [-b ] [--clear-prefs] [-f ] [-g ] [-h] [-l ] [-m ] [-n ] [--no-splash] [-o ] [-s ] [-t ] [--toplevel-circuit ] [-u ] [-v] [-w ]

Les options prises en charge sont les suivantes.

-u, --user-template [empty|plain|filename]

Configure le modèle à utiliser par Logisim. "empty" pour un modèle vide, "plain" pour le modèle par défaut, ou un nom de fichier pour un modèle a charger.

-g, --gates [ansi|iec]

Configure le style de symbole de porte à utiliser.

-o, --locale LocaleIdentifier

Configure la traduction à utiliser. Au moment d'écrire ces lignes, les paramètres régionaux pris en charge incluent:

de Allemand
el Grec
en Anglais
es Espagnol
fr Français
pt Portugais
ru Russe
it Italien
nl Néerlandais
jp Japonais
--clear-prefs

Effacer toutes les préférences de l'application au démarrage, Logisim agira comme s'il était exécuté sur le système hôte pour la première fois.

--no-splash

Masque l'écran de démarrage de Logisim.

-h, --help

Affiche un résumé des options de la ligne de commande.

-v, --version

Affiche le numéro de version de Logisim et s'arrète.

-l, --load filename

Charge une image dans les RAM (fonctionne seulement avec -tty). Plus d'information dans Autres options de vérifications

-s, --substitute file1 file2

Charger le fichier en remplaçant le fichier de bibliothèque (file1) par le le fichier de bibliothèque (file2). Plus d'information dans Substituer des librairies

-t, --tty [table|speed|tty|halt|stats|binary|hex|csv|tabs]

Exécuter sans interface graphique. Plus d'information dans Autres options de vérifications

-w, --test-vector circuitname vectorfile projectfile
Exécute les tests de circuitname dans projectfile en focntion des vecteurs dans vectorfile. Plus d'information dans Fenêtre Vecteur de test
-m, --geometry [widthxheight|widthxheight+X+Y]

Ouvre Logisim avec une fenêtre dimensionnée selon les paramètres W (largeur) et H (hauteur) ou avec l'angle supérieur gauche positioné selon les paramètres X,Y.
Exemple: -geom 600x400+100+100

-n, --new-file-format circ_input circ_output

OUvre un fichier .circ (circ_input) d'une version de logisim et l'enregistre dans un nouveau fichier (circ_output) au format courant utiliseé par Logisisim-evolution.

Options non documentées

-b, --test-circuit testCircuitPathInput
Open up a circ file and start the test bench within it. It returns Success or fail.
-f, --test-fpga inputpath Circuitimapfile cicuitimpname cicuitimpboard
Test implementations design using circ_input file. The implementation circuit_name is the circuit choose to implement(FPGA_Top) and the board is the board name located in resources/logisim/boards/ without the ".xml" extension. The optional tick frequency is in Hz. The optional switch HDLONLY will only generate the HDL-description and will not perform synthesis, P&R, and Download. Arguments: [tick frequency] [HDLONLY]
-test-circ-gen testCircPathInput testCircPathOutput
-questa [yes|no]
